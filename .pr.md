## Design Note
- Added `portfolio/uploads.py` with byte-signature validators that reject unexpected file types and enforce clear size caps.
- Added a `Submission` model and a submission form so the public contact page can accept avatar and PDF uploads safely.
- Stored uploaded files under a private `MEDIA_ROOT` and served them only through staff-authenticated download views.
- Added a staff-only review page so private submissions can be inspected without exposing the files to the public site.

## Security Impact
- **Unsafe File Types Are Blocked**: SVGs, mismatched extensions, and unexpected content signatures are rejected before save.
- **Size Limits Are Enforced**: Avatar and document uploads have explicit maximum sizes so oversized files fail validation.
- **Uploaded Files Stay Private**: Files are written to a private storage root instead of a public media endpoint.
- **Download Time Is Controlled**: Staff-only download views set safe content-disposition headers and `nosniff` to reduce browser-side risks.
