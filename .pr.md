## Design Note
- Added a dedicated charles/view_profile/<int:pk>/ route and a secure view_profile view.
- The view verifies the requesting user either owns the profile or belongs to the instructor group before returning data.
- Unauthorized requests raise PermissionDenied before get_object_or_404 to avoid resource existence leakage.
- Existing profile update and account-management flows remain unchanged.

## Security Impact
- Mitigates Broken Access Control / IDOR by validating object ownership explicitly.
- Removes unsafe assumptions that login status alone is sufficient.
- Prevents users from viewing other users' profile data by changing the URL identifier.
- Uses 403 Forbidden for unauthorized access and 404 Not Found only for valid principals requesting non-existent resources.
