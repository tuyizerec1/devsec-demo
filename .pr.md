## Assignment Summary
- Secured avatar and document uploads by validating file type and size, storing uploads in a private directory, and serving them only through login-protected download views.

## Related Issue
- Closes #41

## Target Assignment Branch

## Design Note
- Added `charles/uploads.py` with upload validators that inspect file signatures instead of trusting client-provided MIME data alone.
- Allowed PNG, JPEG, GIF, and WEBP avatars plus PDF documents only, with 2 MB and 5 MB size limits respectively.
- Stored uploaded files under a private on-disk root and exposed them only through authenticated profile download views.
- Kept the profile page usable by separating normal bio editing from the file-upload form.

## Security Impact
- **Unsafe File Types Are Rejected**: SVGs, mismatched extensions, and other unexpected uploads are blocked before they are saved.
- **Size Limits Are Enforced**: Avatar and document uploads have clear maximum sizes so large files are rejected early.
- **Uploaded Content Is Not Publicly Served**: Files are not exposed through a raw media endpoint; users must be logged in to retrieve them.
- **Download Handling Is Controlled**: Avatar files are served inline only from the authenticated view, and documents are delivered as attachments.
