## Design Note
- Implemented a small `LoginAttempt` model that tracks consecutive failures by normalized username.
- Set a threshold of 5 failed attempts before triggering a 15-minute temporary lockout.
- Chose account-based throttling because it is simple, auditable, persists cleanly in Django's database layer, and is easy to test without relying on cache configuration.
- Accepted the tradeoff that account-based throttling can be abused for temporary denial of service against a known username, but limited the impact with an automatic expiry window and clear user feedback.

## Security Impact
- **Brute-Force Resistance**: Repeated password guessing against the same account is slowed after 5 consecutive failures.
- **User Enumeration Safety**: Attempt records are keyed to submitted usernames without changing the existing generic login error behavior.
- **Usability Balance**: Lockouts expire automatically after 15 minutes and the user sees a clear wait-time message instead of a permanent block.

## Changes Made
- Added a `LoginAttempt` model and migration to track failed logins by normalized username.
- Updated `charles/views.py` to enforce a 5-attempt threshold, deny logins during the lockout window, reset expired lockouts, and clear attempts on successful login.
- Registered `LoginAttempt` in Django admin for visibility during review and debugging.
- Added 10 brute-force regression tests covering normal login behavior, lockout enforcement, expiry, reset, case normalization, and per-account isolation.

## Validation
- **Unit Testing**: 37 tests pass total (27 existing + 10 brute-force tests).
- Verified that valid logins still work and redirect normally.
- Verified that 4 failures do not lock an account, but the 5th failure does.
- Verified that a locked account rejects even the correct password until the lockout expires.
- Verified that the lockout message includes wait-time guidance for legitimate users.
- Verified that successful login clears prior failed-attempt records.
- Verified that expired lockouts reset cleanly and that one account's failures do not affect another account.
