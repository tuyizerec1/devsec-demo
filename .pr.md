## Design Note
- Created a dedicated Django app named "charles" extending Django's built-in User model with a Profile model via OneToOneField.
- Implemented complete authentication lifecycle: registration, login, logout, password change, and profile management.
- Used Django's built-in auth forms and decorators to avoid custom security mistakes.
- Applied secure defaults: HTTPONLY session cookies, 1-hour session timeout, open-redirect protection, POST-only logout.

## Security Impact
- Secure user authentication with Django's built-in hashing and validation.
- Protected forms with CSRF tokens on all POST endpoints.
- Session security with HTTPONLY cookies to prevent XSS theft of session data.
- Open-redirect protection on login with url_has_allowed_host_and_scheme validation.
- POST-only logout to prevent CSRF-style silent sign-out attacks.
- Password changes preserve session with update_session_auth_hash.
- User enumeration prevented by generic auth error messages.
